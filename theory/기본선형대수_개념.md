# Three.js를 위한 기본 선형대수

이 문서는 `vite-project/src/ts/c_1.ts`에 나온 코드만 이해하는 것을 목표로 한다.  
처음에는 수식보다 **화면에서 물체가 어디에 있고, 어느 방향으로 움직이는지 표현하는 방법**으로 생각하면 된다.

## 0. 먼저 알아둘 점: 현재 예제는 화면에 아무것도 그리지 않는다

`c_1.ts`는 벡터를 계산하고 `console.log()`로 결과를 출력하는 코드다. `Scene`, `Camera`, `WebGLRenderer`가 없고 `group`을 씬에 넣지도 않았기 때문에 브라우저 화면이 비어 있는 것이 정상이다.

Three.js에서 물체를 보려면 보통 다음 흐름이 필요하다.

```text
Mesh(물체) → Scene(공간) → Camera(눈) → Renderer(브라우저 화면)
```

이 문서의 뒤쪽 `group` 예제는 이 흐름을 만들기 전의 **좌표 계산 연습**이다.

---

## 0-1. 예제에 나온 Three.js 물체 네 가지

```ts
const group = new Three.Group();

const child = new Three.Mesh(
  new Three.BoxGeometry(),
  new Three.MeshBasicMaterial({ color: 0x00ff00 }),
);
```

이 코드는 "초록색 큐브를 만들고, 그룹 안에 넣을 준비"를 하는 코드다.

| 코드 | 쉬운 비유 | 역할 |
| --- | --- | --- |
| `Group()` | 투명한 묶음 상자 | 여러 물체를 한 번에 이동·회전·확대할 수 있게 묶는다. 화면에 보이는 모양은 없다. |
| `BoxGeometry()` | 큐브의 뼈대/모양 | 큐브의 꼭짓점과 면이 어디에 있는지 정한다. 색은 없다. |
| `MeshBasicMaterial()` | 표면의 색칠 | 모양의 표면을 어떤 색으로 보이게 할지 정한다. 빛 영향을 받지 않아 조명 없이도 색이 보인다. |
| `Mesh()` | 완성된 실제 물체 | `Geometry`(모양)와 `Material`(표면)을 합친, 씬에 넣어 그릴 수 있는 물체다. |

관계는 다음과 같다.

```text
BoxGeometry(큐브 모양) + MeshBasicMaterial(초록 표면)
                         ↓
                    Mesh(초록 큐브)
                         ↓
                Group(큐브를 담는 투명한 상자)
```

그래서 다음 코드는 큐브를 그룹의 자식으로 넣는다는 의미다.

```ts
group.add(child);
```

이후 `group.rotation.y`를 돌리면, 그룹 안의 `child`도 함께 회전한다.

## 1. `Vector3` = x, y, z 세 숫자

```ts
const v = new Three.Vector3(x, y, z);
```

`Vector3`는 세 방향의 값을 한 묶음으로 저장한 객체다.

```text
x: 오른쪽(+) / 왼쪽(-)
y: 위(+)     / 아래(-)
z: 화면 앞(+) / 화면 뒤(-)
```

예를 들어 다음은 x축, 즉 오른쪽 방향을 가리킨다.

```ts
const a = new Three.Vector3(1, 0, 0);
```

```text
a = (1, 0, 0)

원점 ● ──→ 오른쪽 1칸
```

다음은 y축, 즉 위쪽 방향이다.

```ts
const b = new Three.Vector3(0, 1, 0);
```

```text
b = (0, 1, 0)

     ↑ 위로 1칸
     ●
```

### 위치와 방향은 둘 다 `Vector3`를 쓴다

같은 `(1, 0, 0)`이라도 문맥에 따라 의미가 다르다.

- **위치**: 원점에서 오른쪽으로 1칸 떨어진 장소
- **방향/이동량**: 오른쪽으로 1칸 가라는 화살표

Three.js는 둘 다 `Vector3`로 표현한다. 코드가 `position`에 들어가면 위치, `direction`이나 속도 계산에 쓰이면 방향이라고 보면 된다.

## 1-1. `Vector3`에서 자주 쓰는 네 함수

아래 두 벡터를 기준으로 보자.

```ts
const a = new Three.Vector3(1, 0, 0); // 오른쪽으로 1
const b = new Three.Vector3(0, 1, 0); // 위로 1
```

### `clone()` — 복사본 만들기

```ts
const copyA = a.clone();
```

`a`와 같은 값을 가진 **새로운** `Vector3`를 만든다.

```text
a     = (1, 0, 0)
copyA = (1, 0, 0)  // 값은 같지만 a와 별개
```

왜 필요할까? `add()`나 `normalize()`처럼 원본 벡터를 직접 바꾸는 함수가 있기 때문이다. 원래 값을 남겨 두고 계산하려면 `clone()`부터 한다.

### `add()` — 두 이동량을 더하기

```ts
const sum = a.clone().add(b);
```

```text
a 복사본: (1, 0, 0)
b:         (0, 1, 0)
결과:      (1, 1, 0)
```

중요: `add()`는 호출한 쪽을 바꾼다.

```ts
a.add(b);
console.log(a); // (1, 1, 0) — a가 바뀜
```

그래서 원본 `a`를 지키려면 `a.clone().add(b)`라고 쓴다.

### `distanceTo()` — 두 위치 사이의 거리

```ts
const distance = a.distanceTo(b);
```

`a`와 `b`라는 두 지점 사이의 직선거리를 구한다. 이 함수는 `a`와 `b`를 바꾸지 않는다.

```text
a = (1, 0, 0)
b = (0, 1, 0)
distance = 약 1.414
```

예를 들어 플레이어와 적이 가까운지 확인할 때 쓸 수 있다.

```ts
if (player.position.distanceTo(enemy.position) < 3) {
  // 두 물체가 3보다 가까움
}
```

### `normalize()` — 방향은 유지하고 길이를 1로 맞추기

```ts
const direction = new Three.Vector3(1, 1, 0).normalize();
```

`(1, 1, 0)`은 오른쪽 위 방향이다. `normalize()` 후에도 방향은 같지만, 화살표 길이가 정확히 1이 된다.

```text
전: (1, 1, 0)          길이 약 1.414
후: (0.707, 0.707, 0)  길이 정확히 1
```

`normalize()`도 원본을 바꾼다. 대각선 이동이 가로 이동보다 빨라지지 않도록, 이동 전에 방향을 정리할 때 주로 쓴다.

```ts
const direction = new Three.Vector3(1, 1, 0).normalize();
cube.position.add(direction.multiplyScalar(0.1));
```

### 한눈에 보기

| 함수 | 하는 일 | 원본을 바꾸나? |
| --- | --- | --- |
| `clone()` | 같은 값을 가진 새 벡터를 만든다 | 아니오 |
| `add(v)` | 현재 벡터에 `v`를 더한다 | 예 |
| `distanceTo(v)` | 현재 위치와 `v` 사이의 거리를 구한다 | 아니오 |
| `normalize()` | 방향은 유지하고 길이를 1로 만든다 | 예 |

## 2. 벡터 덧셈: 이동을 이어 붙이기

```ts
const sum = a.clone().add(b);
```

이 코드는 "오른쪽으로 1칸"과 "위로 1칸"을 합친다.

```text
a = (1, 0, 0)  오른쪽 1칸
b = (0, 1, 0)  위로 1칸

sum = (1 + 0, 0 + 1, 0 + 0)
    = (1, 1, 0)
```

화면의 2차원 그림으로 보면 오른쪽 위 대각선으로 이동하는 것이다.

```text
(0, 0) ● ──→ (1, 0)
                 │
                 ↑
               (1, 1)
```

### 왜 `clone()`을 붙일까?

Three.js의 `add()`는 계산 결과를 새로 돌려주는 대신, **호출한 벡터 자신을 바꾼다.**

```ts
a.add(b); // a 자체가 (1, 1, 0)으로 바뀜
```

원래의 `a = (1, 0, 0)`을 보존하고 싶다면 복사본에 더한다.

```ts
const sum = a.clone().add(b); // a는 그대로, sum만 (1, 1, 0)
```

## 3. 거리: 두 위치 사이의 직선 길이

```ts
const dist = a.distanceTo(b);
```

`distanceTo()`는 `a`와 `b`라는 두 점 사이의 최단거리, 즉 직선 길이를 구한다.

```text
(0, 1) b ●
         ╲
          ╲  dist
           ╲
            ● a (1, 0)
```

이 예제의 결과는 약 `1.414`다. 수식으로는 `√2`이지만, 처음에는 **"두 물체가 얼마나 떨어졌는가"**를 구하는 함수라고 기억하면 충분하다.

실제 활용 예:

```ts
const distance = player.position.distanceTo(enemy.position);
if (distance < 3) {
  // 적이 플레이어에게 충분히 가까움
}
```

## 4. 정규화(`normalize`): 방향은 유지하고 길이만 1로 만들기

```ts
const normalized = sum.clone().normalize();
```

`sum`은 `(1, 1, 0)`, 즉 오른쪽 위를 향한다. 다만 이 화살표의 길이는 1이 아니라 약 `1.414`다.

`normalize()`는 화살표의 **방향은 그대로 두고 길이만 정확히 1로 바꾼다.**

```text
sum:        ↗  길이 약 1.414
normalized: ↗  길이 정확히 1
```

결과 값은 대략 다음과 같다.

```ts
normalized; // (0.707, 0.707, 0)
```

여기서 `0.707`이라는 **각 숫자**가 1이라는 뜻은 아니다. `Vector3`에서는 x, y, z 값을 합쳐 만든 **화살표 전체 길이**를 말한다.

```text
(0.707, 0.707, 0)의 길이
= √(0.707² + 0.707² + 0²)
= √(약 0.5 + 약 0.5)
= √1
= 1
```

`(0.808, 0.808, 0)`은 둘 다 더 큰 값이므로 길이가 1이 아니다.

```text
길이 = √(0.808² + 0.808²) ≈ 1.143
```

대각선 `(1, 1, 0)`의 길이를 정확히 1로 줄일 때 각 값은 `1 ÷ √2`, 즉 약 `0.707106...`이 된다.

왜 필요할까? 대각선 이동을 그대로 더하면 가로 이동보다 더 빠를 수 있기 때문이다. 먼저 방향을 정규화한 뒤 속도를 곱하면, 어느 방향이든 같은 속도로 움직일 수 있다.

```ts
const direction = new Three.Vector3(1, 1, 0).normalize();
const speed = 5;

mesh.position.add(direction.multiplyScalar(speed * deltaTime));
```

## 5. 부모와 자식: 로컬 좌표와 월드 좌표

```ts
const group = new Three.Group();
const child = new Three.Mesh(/* 큐브 */);

child.position.set(2, 0, 0);
group.add(child);
```

`group`은 물체를 담는 투명한 상자라고 생각하면 된다. `child`는 그 상자 안의 초록색 큐브다.

```text
group 안

중심 ● ──────■ child
       오른쪽 2칸
```

### `child.position`은 부모 기준 위치다

```ts
child.position; // (2, 0, 0)
```

이는 "세계의 원점에서 오른쪽 2칸"이라는 뜻이 아니다.  
**`group`의 중심을 기준으로 오른쪽 2칸**이라는 뜻이다. 이런 좌표를 **로컬 좌표(local position)**라고 한다.

### 부모를 돌리면 자식도 함께 돈다

```ts
group.rotation.y = Math.PI / 4; // y축 기준 45도 회전
```

상자(`group`)를 돌리면 그 안에 붙어 있는 큐브도 같이 돈다. 큐브가 상자 안에서 차지하는 자리 자체는 여전히 "오른쪽 2칸"이므로 `child.position`은 바뀌지 않는다.

하지만 세계 전체에서 보면 큐브의 실제 위치는 회전한 방향으로 바뀐다. 이 실제 위치가 **월드 좌표(world position)**다.

```ts
child.getWorldPosition(new Three.Vector3());
// 대략 (1.414, 0, -1.414)
```

정리:

| 코드 | 질문 | 이 예제의 의미 |
| --- | --- | --- |
| `child.position` | 부모 `group` 기준으로 어디에 있나? | 오른쪽 2칸, `(2, 0, 0)` |
| `child.getWorldPosition(...)` | 세계 전체 기준으로 실제 어디에 있나? | 그룹 회전까지 적용된 위치 |

부모의 `position`, `rotation`, `scale`은 모두 자식의 월드 위치와 모양에 영향을 준다.

---

## 이 예제에서 꼭 기억할 4문장

1. `Vector3(x, y, z)`는 3차원 위치나 방향을 담는 세 숫자다.
2. `add()`는 이동/방향을 합치며, 원본 벡터를 바꾼다. 보존하려면 `clone()`한다.
3. `distanceTo()`는 두 위치 사이 거리, `normalize()`는 방향은 유지하고 길이를 1로 만든다.
4. `position`은 부모 기준(로컬)이고, `getWorldPosition()`은 부모 변환까지 반영한 실제 세계 기준(월드) 위치다.
